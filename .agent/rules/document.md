# Documentation Standards

## General Requirements
- All documents must use Markdown format
- Use concise and clear language
- Keep documentation up to date
- Avoid spelling and grammatical errors
- Use Chinese as the primary language

## Directory Structure
- `README.md`: Located in the project root, provides project overview
- `docs/`: Contains detailed documentation
  - `guide/`: User guides
  - `api/`: API documentation
  - `examples/`: Example code documentation

## README.md Content Standards
- Project name and brief description
- Technology stack overview
- Explanation of project structure
- Usage instructions
- License information

## Changelog Standards
- Use `CHANGELOG.md` to record version changes
- Follow Semantic Versioning guidelines
- Each version should include: new features, bug fixes, breaking changes

## Documentation Content Organization
- Organize from general to specific, simple to complex
- Place important information at the top
- Group related content together
- Use subheadings and lists to enhance readability
- Avoid long paragraphs; keep content concise

## Code Example Standards
- Provide complete, runnable examples
- Code should be concise and easy to understand
- Add appropriate comments to explain key parts
- Specify the expected output or behavior of the code
- Update examples to match the latest API
